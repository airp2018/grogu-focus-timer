# 2026-07-03 工作日志

## Grogu Focus Timer 扩展设计评审
对 `extension/` 目录下的曼达洛人主题专注计时器 Chrome 扩展做了完整设计评审。

### 核心发现
- **致命 bug**：`background.js` alarm 监听（约 300-314 行）focus 结束后 `timerState.mode` 被重置为 `'focus'` 而非 `'break'`，导致休息模式从未启动，`breakDuration` 配置是死代码，番茄钟工作流断裂。
- **留存缺失**：无番茄计数/统计、无任务归档、无连续天数——番茄钟产品命脉缺失。
- **权限过宽**：`<all_urls>` + `tabs` + `scripting` 组合，且无隐私政策，Chrome Web Store 审核会卡。
- **外部字体**：`popup.css` 第 2 行 `@import` Google Fonts，MV3 CSP 默认不允许，应打包本地或砍掉。
- **ReDoS 风险**：`distractionPatterns` 用户输入直接当正则，无复杂度限制。
- **`activeBlockerTabs` 内存态**：SW 回收后丢失，已注入 overlay 残留。

### 亮点
- 古古拟人化 + NO/YES 情感联动设计有记忆点。
- cradle SVG + drift/peek/alert 三种运动模式 + 原力粒子，动效细节用心。
- offscreen document 播音频方案规范。

### 改进优先级（第一轮，按标准番茄钟评——后已修正）
- P0: 修 break 循环断链；写隐私政策 + 收紧 host_permissions
- P1: 加番茄统计；计时圆环 SVG 进度环；字体本地化
- P2: side panel Tab 分区；正则校验防 ReDoS；召唤面板用 dev flag 隐藏

## 第二轮：用户澄清产品定位后的修正

### 用户真实场景
"20 分钟专注看一个东西，中间有电话/打断，回来继续"——是**灵活专注计时器**，不是标准番茄钟。

### 评审修正
- **break 循环断链不是 bug**：用户不需要自动 5min 休息循环，alarm 重置 mode='focus' 反而符合"单次专注结束，等下一轮"语义。从 P0 降到 P2（清理死代码）。
- **真正的 P0 换成 SW 回收吞时间**：background.js init() 第 93-99 行，targetTime 过期时直接 isRunning=false, timeLeft=0。但"过期"有两种含义被混淆：① 真专注完了 ② SW 被回收（接电话久）。用户接电话回来发现专注"已结束"，剩余时间凭空消失。
- 修法：init() 恢复时改用冻结的 timeLeft，不看 targetTime 过期判定。SW 睡眠期间没在计时，timeLeft 才是"还欠多少专注时间"。

### 新增 P1（贴合场景）
- 自动捕获专注对象：开始专注时抓当前 tab URL+title 存 session，结束时显示"你看完了《XXX》"
- 中断记录：PAUSE_TIMER 时记 {at, duration} 到 interruptions 数组，结束统计"被打断 N 次共 X 分钟，净专注 Y 分钟"

## 第四轮：具体到代码功能/属性要改什么（用户要求不泛泛）

读了 content.js / popup.html / content.css 后，从电子宠物视角列出具体改动点（带行号）：

1. **没有 idle 常驻态（最致命）**：content.js 第 45 行 `root.replaceChildren()` + 第 150-161 行 auto-dismiss 6-11s = 古古只在事件瞬间存在，结束就消失。电子宠物命脉是"它一直在那"。改：新增常驻 idle 小态，事件触发切大态，结束后回 idle 而非消失。
2. **RANDOM_EVENTS 全是"吃"**（background.js 33-38 行）：4 个事件 cookie1/cookie2/meat/eggs 行为谱全一样。改：补翻身/打哈欠/看你一眼又睡回去/伸懒腰/玩原力球/深夜打盹等自发行为。
3. **点击头像只有一种反应**（content.js 120-122 行）：`createForceParticles` 永远喷 15 个粒子。改：按点击位置/次数/连续度映射不同反应（摸头眯眼、戳烦躲开、叫名字歪头）。
4. **NO blocker 说教式**（popup.html 70 行 + content.js 25 行）：`"快回去工作！这不是正道！"` + NO 走 alert motion。改：换 sad motion（缩起低饱和），文案改"古古一个人好无聊…"。
5. **task 死字段**（popup.html 49 / background.js 13）：填了从不用于互动。改：删掉或接进古古状态机（不同 task → idle 姿态不同）。
6. **配置项措辞工具感**（popup.html 69、74 行）："劝阻提示语/启用分心自动劝阻"。改：重命名成陪伴感措辞。
7. **auto-dismiss 太短**（content.js 151-152 行）：主动召唤 7s 就收走。改：主动召唤不自动消失，让用户手动关或长 idle 后回小态。
8. **一次只能一个状态**（content.js 45 行 replaceChildren）：召唤吃马卡龙时触发 YES 会互相覆盖。改：允许状态叠加（边吃边看你）。
